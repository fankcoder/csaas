# cs_web/src/app 提示词

## 路由层目标

`src/app/` 负责页面路由、SEO 元信息和页面级数据消费。每个大功能都应是单独页面。

## 当前页面范围

### 核心转化与展示

- `/`
  - 官网首页，展示品牌、产品价值、样例能力、市场覆盖与升级入口
- `/pricing`
  - 套餐价格与订阅入口
- `/samples`
  - 免费样例套利榜单
- `/login`
  - 登录、注册、邮箱验证码、Steam 入口

### 核心功能

- `/arbitrage`
  - 完整套利分页列表，高级功能，需订阅访问
- `/items/[id]`
  - 单饰品高级详情、历史价格、方向分析
- `/profile`
  - 个人中心、Steam 绑定、库存查询
- `/onboarding`
  - 平台开通清单
- `/favorites`
  - 机会收藏
- `/reports`
  - 用户收益报表

### 内容与 SEO

- `/faq`
- `/guide`
- `/blog`
- `/blog/[slug]`
- `/terms`
- `/privacy`
- `/disclaimer`

## 页面设计要求

### 首页

- 风格要有 CS 世界观与数据战场感
- 明确强调跨中国市场与海外市场的差异化能力
- 让未订阅用户快速理解“样例榜单 + 完整榜单 + 订阅升级”路径

### 数据页

- 列表页必须支持分页
- 筛选条件要贴合后端 query 参数
- 未订阅时要清晰引导用户升级，而不是直接白屏

### 个人页

- 允许查看账户状态、绑定 Steam、拉取库存
- 需要明确显示高级权限状态

### SEO 页面

- 文章和 FAQ 既服务转化，也服务搜索引擎
- 内容应围绕平台教程、套利教育、费用认知、风险提示展开

## 元信息要求

- 所有重要页面都应设置 `title`、`description`
- 文章页与营销页需要可索引
- `robots` 与 `sitemap` 应与公开页面保持一致
