# cs_web/src/components 提示词

## 模块目标

`src/components/` 存放可复用组件、全局外壳、状态展示组件和一些页面组装块。

## 当前组件方向

- `AppShell`
  - 全站导航、页脚、整体布局容器
- `MarketStatusStrip`
  - 展示平台覆盖、刷新状态、汇率等轻量数据
- `Dashboard` / `FVPage`
  - 页面级组合组件或承载复杂展示块
- `LegalDocument`
  - 法律说明类页面排版
- `CookieConsent`
  - 站点基础合规交互

## 设计原则

- 组件优先保持职责单一
- 页面级复杂模块可拆成可复用卡片、列表、统计条、图表块
- 保持移动端与桌面端都可读
- 样式命名和布局方式要与全站视觉语言统一

## 交互要求

- 登录态相关组件要处理“未登录 / 已登录 / 权限不足”三种状态
- 数据组件要有：
  - loading
  - empty
  - error
- 不要把复杂 API 调用直接堆在纯展示组件里

## 视觉要求

- 首页组件可以更大胆、更氛围化
- 内页组件应更稳定、更像数据产品
- 尽量通过统一面板、标签、状态色、数值排版建立品牌一致性
